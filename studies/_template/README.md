# StudyName

One-sentence hook — the "huh, really?" that the study reveals.

## The question

What are you asking? State it in one or two sentences. A reader should be able to read the question and already know whether they care about the answer.

## The method

How do you answer it? Which codec, what inputs, what metric. Three to five sentences. If the method needs more room, move the extra detail to `notes.md`.

## The result

What the numbers or plot actually show. Include the result inline — a table, a small figure, or a few bullet points. Cite the exact command a reader would run to reproduce.

```bash
python -m studies.<name>.run
```

## The conclusion

What does this mean? Three to five sentences. Be honest about the scope of the claim — what the study does and does not show. If the finding is regime-specific, say so.

## Caveats

What the study does not address, where the mechanism might break down, what a critical reader would rightly push back on.

## Prior art

If this study engages with published work, cite it.
